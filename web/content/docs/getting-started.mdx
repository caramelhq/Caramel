## Adding Caramel

Click the invite link to add Caramel to your server. You'll need **Manage Server** permissions on the target guild.

Once Caramel joins, it's completely inactive by default. Nothing runs until you explicitly set up and enable a module. This is by design — Caramel only does what you tell it to.

## Required Permissions

Caramel needs the following base permissions to function correctly:

| Permission       | Why it's needed                            |
| ---------------- | ------------------------------------------ |
| Manage Roles     | Assigning muted roles and vanity roles     |
| Manage Channels  | Creating private log channels during setup |
| Kick Members     | `/mod kick` command                        |
| Ban Members      | `/mod ban` and `/mod softban` commands     |
| Moderate Members | `/mod timeout` and `/mod unmute` commands  |
| Manage Messages  | Deleting silent-banned users' messages     |
| Send Messages    | Posting mod logs and lockdown notices      |
| View Channels    | Accessing channels for moderation          |

Make sure Caramel's role is positioned **above** any roles you want it to be able to manage (e.g., Muted roles, vanity roles).

## Setting Up Your First Module

Caramel uses a **module system** — nothing is active by default. You choose what to enable and configure.

There are two ways to set up modules: via the **web dashboard** or directly via **Discord slash commands**. Both are fully supported.

### Option A: Web Dashboard

1. Visit the dashboard and log in with your Discord account
2. Select your server from the server picker
3. Navigate to the **Modules** page
4. Click the toggle next to a module to enable it
5. Click the module card to expand its configuration settings
6. Select your log channel, muted role, and other options from the dropdowns
7. Click **Save**

The dashboard uses a live channel and role picker — you'll see every text channel and role in your server listed by name, no IDs required.

### Option B: Discord Slash Commands

**Step 1 — Run setup**

```
/module setup name:Moderation
```

This opens an interactive modal. Fill in:

- **Log channel** — paste a channel ID or name, or leave empty to auto-create `#mod-logs`
- **Muted role** — paste a role ID or name, or leave empty to auto-create a `Muted` role

If you leave either field empty, Caramel creates the channel or role automatically and links it. You'll see a confirmation summary before anything is created — click **Confirm** to proceed or **Cancel** to abort.

**Step 2 — Enable the module**

```
/module enable name:Moderation
```

Caramel runs a validation check before enabling. If anything is missing (e.g., the configured role was deleted), it will tell you exactly what needs to be fixed.

**Step 3 — Verify the configuration**

```
/module settings name:Moderation
```

This displays the current configuration for the module: log channel, muted role, threshold settings, and whether the module is active.

## Available Modules

| Module             | What it does                                                                 |
| ------------------ | ---------------------------------------------------------------------------- |
| **Moderation**     | warn, mute, ban, softban, kick, timeout, unmute, slowmode, lockdown, history |
| **Vanity Tracker** | Auto-assigns a role to members who include a keyword in their custom status  |

## Message Prefix

In addition to slash commands, every moderation command works with the `c!` prefix:

```
/mod ban @user reason:Spamming
c!mod ban @user Spamming
```

```
/mod mute @user duration:1h reason:Disruptive
c!mod mute @user 1h Disruptive
```

The prefix is `c!` by default. This cannot currently be changed via a command — both `c!` and the slash command variants always work.
