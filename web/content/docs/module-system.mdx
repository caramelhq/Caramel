## Overview

Every feature in Caramel is packaged as an **independent module**. Nothing runs by default — you decide what to activate for your server. Modules are fully isolated: enabling, disabling, or resetting one has zero effect on any other module or on any other server.

## Available Modules

| Module             | Requires Setup                 | Key Features                                                                             |
| ------------------ | ------------------------------ | ---------------------------------------------------------------------------------------- |
| **Moderation**     | Yes (log channel + muted role) | warn, mute, ban, softban, kick, timeout, unmute, slowmode, lockdown, history, silent ban |
| **Vanity Tracker** | Yes (keyword + role)           | Auto-assign/revoke a role based on custom status keyword detection                       |

## Module Lifecycle

Every module follows the same lifecycle:

```
Setup → Validate → Enable → (use) → Disable / Reset
```

1. **Setup** — configure the module's settings (channels, roles, keywords, etc.)
2. **Validate** — Caramel checks that all required configuration is present before enabling
3. **Enable** — activate the module for your server
4. **Disable** — turn off the module without losing configuration
5. **Reset** — factory reset: wipes all config and disables the module

## Managing Modules

You can manage modules from the **web dashboard** or via Discord slash commands. Both methods are fully equivalent.

### Via the Web Dashboard

1. Log in and select your server at `/dashboard`
2. Click **Modules** in the sidebar
3. Use the toggle on each module card to enable or disable
4. Click a module card to expand its configuration panel
5. Configure channels, roles, and other settings using the live pickers
6. Click **Save** to persist changes

### Via Discord Slash Commands

All module commands require **Administrator** permission.

#### Setup

Open an interactive modal to configure a module's settings:

```
/module setup name:Moderation
/module setup name:Vanity Tracker
```

The modal collects all required settings for the module. You can provide existing channel/role IDs or names, or leave fields empty and Caramel will auto-create the necessary resources.

After submitting the modal, a **confirmation preview** is shown listing exactly what will be created or linked. Click **Confirm** to apply or **Cancel** to abort. The confirmation times out after 30 seconds.

#### Enable

Activate a module for your server:

```
/module enable name:Moderation
/module enable name:Vanity Tracker
```

Before activating, Caramel runs **pre-enable validation** (see below). The module only activates if all checks pass.

#### Disable

Deactivate a module without losing any of its configuration:

```
/module disable name:Moderation
/module disable name:Vanity Tracker
```

The module stops functioning immediately, but all settings are preserved. You can re-enable it at any time with `/module enable`.

#### Settings

View the current configuration for a module:

```
/module settings name:Moderation
/module settings name:Vanity Tracker
```

Shows all configured values: channels, roles, thresholds, whether the module is active, and whether each resource was created by Caramel or linked from an existing one.

#### Reset

Factory reset a module — clears all configuration, disables the module, and optionally deletes any resources Caramel created during setup:

```
/module reset name:Moderation
/module reset name:Vanity Tracker
```

A confirmation prompt is shown first, listing exactly what will be deleted or unlinked:

- **Created by Caramel** (e.g., auto-created `#mod-logs` channel or `Muted` role) → **deleted**
- **Linked by you** (existing channels/roles you provided) → **unlinked only**, not deleted

The confirmation times out after 20 seconds.

## Pre-Enable Validation

Before a module can be enabled, Caramel runs a validation pass to ensure all required configuration exists and is still valid.

**Moderation** requires:

- A valid mod log channel (exists in the server)
- A valid muted role (exists in the server)

**Vanity Tracker** requires:

- A keyword configured
- A valid vanity role (exists in the server)
- A valid log channel (exists in the server)

If validation fails, the bot responds with a specific list of what's missing. This can happen if, for example, a role or channel was deleted after the initial setup. Run `/module setup` again to reconfigure, then re-enable.

## Per-Guild Isolation

Every server has its own independent module configuration. Caramel stores all config per `guildId` in the database:

- Server A can have Moderation enabled and Vanity Tracker disabled
- Server B can have both enabled with completely different configurations
- Resetting a module in one server has no effect on any other server

This also means Caramel never leaks configuration between servers, even if the same moderator manages multiple servers.
