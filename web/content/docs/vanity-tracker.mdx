title: Vanity Tracker
description: Automatically assign and revoke roles based on custom status keywords. Powered by async BullMQ job queues — zero latency on your commands.

## Overview

The Vanity Tracker module monitors your members' custom Discord statuses for a specific keyword. When a member adds the keyword to their status, they're automatically assigned the configured vanity role. When they remove it, the role is revoked.

All status checks run **asynchronously** via BullMQ job queues — no synchronous blocking, zero delay on commands, and no missed updates even under high member traffic.

## Common Use Cases

- **Vanity URL promotion** — reward members who display your server's Discord vanity URL (`discord.gg/yourserver`) in their status
- **Community events** — track members showing a specific event tag or hashtag in their status
- **Brand representation** — incentivize members to represent your server or community

## Setup

### Via the Web Dashboard

1. Go to **Dashboard > Modules**
2. Toggle **Vanity Tracker** on
3. Navigate to **Settings**
4. Enter your vanity keyword, select a role from the dropdown, and optionally select a log channel
5. Click **Save**

### Via Discord

```
/module setup name:Vanity Tracker
/module enable name:Vanity Tracker
```

Running `/module setup name:Vanity Tracker` opens a modal with three fields:

| Field              | Required | Description                                                                                                     |
| ------------------ | -------- | --------------------------------------------------------------------------------------------------------------- |
| Status keyword     | Yes      | The text to detect in member custom statuses. Case-insensitive.                                                 |
| Role ID or name    | No       | The role to assign. Paste an ID, a role name, or leave empty to auto-create.                                    |
| Channel ID or name | No       | Log channel for status change notifications. Paste an ID, a name, or leave empty to auto-create `#vanity-logs`. |

If you leave the role or channel fields empty, Caramel creates them automatically. A confirmation step shows exactly what will be created — click **Confirm** to apply or **Cancel** to abort.

**Example setup values:**

- Keyword: `discord.gg/myserver`
- Role: leave empty (Caramel creates `Vanity Role [Server Name]`)
- Channel: leave empty (Caramel creates `#vanity-logs`)

## How It Works

1. A member updates their custom Discord status
2. Caramel receives the `PresenceUpdate` event
3. The new status is compared against the configured keyword
4. If the keyword is **added** → a job is queued to assign the vanity role
5. If the keyword is **removed** → a job is queued to revoke the vanity role
6. The BullMQ worker processes the job and applies the role change
7. A notification is posted to the configured log channel (if set)

When a new member joins the server, Caramel also queues a vanity check for them (with a 2-second delay) in case they already have the keyword in their status.

## Pre-Enable Validation

Before the Vanity Tracker module can be enabled, Caramel validates that:

- A keyword has been configured
- The vanity role exists in the server (if an ID was provided)
- The log channel exists in the server (if an ID was provided)

If any of these checks fail, the bot will tell you exactly what's missing. Fix the issue and run `/module enable name:Vanity Tracker` again.

## Resetting the Module

```
/module reset name:Vanity Tracker
```

A factory reset clears all Vanity Tracker configuration and disables the module. If Caramel created the vanity role or log channel during setup, those resources will also be **deleted**. If you linked existing ones, they are only unlinked — not deleted.

You'll see a confirmation prompt listing exactly what will be removed before anything happens.
