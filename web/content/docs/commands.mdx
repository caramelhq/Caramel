## Overview

All commands are available as both **slash commands** (type `/`) and **prefix commands** (type `c!`). Slash commands show autocompletion and option hints in Discord. Prefix commands are faster to type for experienced moderators.

Moderation commands (`/mod`) require the **Moderate Members** permission. Module commands (`/module`) require **Administrator**.

---

## Moderation Commands — `/mod`

### `/mod warn`

Issue a formal warning to a member.

| Option   | Type         | Required | Description            |
| -------- | ------------ | -------- | ---------------------- |
| `user`   | User mention | Yes      | Member to warn         |
| `reason` | Text         | No       | Reason for the warning |

```
/mod warn user:@SpamBot reason:Sending unsolicited DMs
c!mod warn @SpamBot Sending unsolicited DMs
```

---

### `/mod mute`

Mute a member by assigning the configured muted role. Supports optional timed duration.

| Option     | Type         | Required | Description                       |
| ---------- | ------------ | -------- | --------------------------------- |
| `user`     | User mention | Yes      | Member to mute                    |
| `duration` | Text         | No       | e.g. `30m`, `2h`, `1d`, `1d2h30m` |
| `reason`   | Text         | No       | Reason for the mute               |

```
/mod mute user:@Troll duration:2h reason:Repeated spamming
c!mod mute @Troll 2h Repeated spamming
```

Duration format: combine `d` (days), `h` (hours), `m` (minutes). Examples: `30m`, `6h`, `1d`, `1d12h`.

---

### `/mod timeout`

Apply Discord's native timeout to a member.

| Option     | Type         | Required | Description                          |
| ---------- | ------------ | -------- | ------------------------------------ |
| `user`     | User mention | Yes      | Member to timeout                    |
| `duration` | Text         | No       | e.g. `30m`, `2h`, `1d` (max 28 days) |
| `reason`   | Text         | No       | Reason                               |

```
/mod timeout user:@User duration:30m reason:Disruptive in voice
c!mod timeout @User 30m Disruptive in voice
```

---

### `/mod unmute`

Remove a Discord timeout from a member and clear them from active mute tracking.

| Option | Type         | Required | Description      |
| ------ | ------------ | -------- | ---------------- |
| `user` | User mention | Yes      | Member to unmute |

```
/mod unmute user:@User
c!mod unmute @User
```

---

### `/mod ban`

Permanently ban a member from the server.

| Option        | Type          | Required | Description                                    |
| ------------- | ------------- | -------- | ---------------------------------------------- |
| `user`        | User mention  | Yes      | Member to ban                                  |
| `reason`      | Text          | No       | Reason for the ban                             |
| `delete_days` | Integer (0–7) | No       | Days of message history to delete (default: 0) |

```
/mod ban user:@User reason:Doxxing delete_days:7
c!mod ban @User Doxxing
```

---

### `/mod softban`

Ban and immediately unban a member to purge their recent messages without a permanent ban.

| Option        | Type          | Required | Description                             |
| ------------- | ------------- | -------- | --------------------------------------- |
| `user`        | User mention  | Yes      | Member to softban                       |
| `reason`      | Text          | No       | Reason                                  |
| `delete_days` | Integer (0–7) | No       | Days of messages to delete (default: 3) |

```
/mod softban user:@Spammer reason:Message cleanup delete_days:3
c!mod softban @Spammer Message cleanup
```

---

### `/mod kick`

Remove a member from the server. They can rejoin via an invite.

| Option   | Type         | Required | Description    |
| -------- | ------------ | -------- | -------------- |
| `user`   | User mention | Yes      | Member to kick |
| `reason` | Text         | No       | Reason         |

```
/mod kick user:@User reason:Violating server rules
c!mod kick @User Violating server rules
```

---

### `/mod silentban`

Manage silent bans — stealth restrictions that prevent a user from messaging or joining voice without notifying them.

| Option     | Type         | Required | Description                                           |
| ---------- | ------------ | -------- | ----------------------------------------------------- |
| `action`   | Choice       | Yes      | `Add`, `Remove`, or `List`                            |
| `user`     | User mention | No       | Target user (not needed for `List`)                   |
| `duration` | Choice       | No       | `30m`, `1h`, `6h`, `1d`, `7d`, `30d` (for `Add` only) |
| `reason`   | Text         | No       | Internal reason (for `Add` only, not shown to user)   |

```
/mod silentban action:Add user:@User duration:7d reason:Suspected alt
/mod silentban action:Remove user:@User
/mod silentban action:List
c!mod silentban add @User 7d Suspected alt
c!mod silentban remove @User
c!mod silentban list
```

---

### `/mod slowmode`

Set the slowmode delay for a channel. Use `0` to disable.

| Option    | Type              | Required | Description                           |
| --------- | ----------------- | -------- | ------------------------------------- |
| `seconds` | Integer (0–21600) | Yes      | Slowmode in seconds. `0` disables it. |
| `channel` | Channel           | No       | Target channel (defaults to current)  |

```
/mod slowmode seconds:10
/mod slowmode seconds:0 channel:#general
c!mod slowmode 10
c!mod slowmode 0 #general
```

---

### `/mod lockdown`

Toggle lockdown on a channel. Locks prevent `@everyone` from sending messages. Running again unlocks.

| Option    | Type    | Required | Description                                  |
| --------- | ------- | -------- | -------------------------------------------- |
| `channel` | Channel | No       | Channel to lock/unlock (defaults to current) |

```
/mod lockdown
/mod lockdown channel:#announcements
c!mod lockdown
c!mod lockdown #announcements
```

---

### `/mod history`

View the last 10 moderation actions for a user. Accepts a mention or a raw user ID (for users who left the server).

| Option    | Type         | Required     | Description                    |
| --------- | ------------ | ------------ | ------------------------------ |
| `user`    | User mention | One of these | Member currently in the server |
| `user_id` | Text         | One of these | Raw Discord user ID            |

```
/mod history user:@User
/mod history user_id:123456789012345678
c!mod history @User
c!mod history 123456789012345678
```

## Module Commands — `/module`

All module commands require **Administrator** permission.

### `/module setup`

Open an interactive modal to configure a module. After submitting, shows a confirmation preview before making any changes.

| Option | Type   | Required | Description                      |
| ------ | ------ | -------- | -------------------------------- |
| `name` | Choice | Yes      | `Moderation` or `Vanity Tracker` |

```
/module setup name:Moderation
/module setup name:Vanity Tracker
```

---

### `/module enable`

Enable a module after it has been configured. Runs pre-enable validation before activating.

| Option | Type   | Required | Description                      |
| ------ | ------ | -------- | -------------------------------- |
| `name` | Choice | Yes      | `Moderation` or `Vanity Tracker` |

```
/module enable name:Moderation
/module enable name:Vanity Tracker
```

---

### `/module disable`

Disable a module without clearing its configuration. Can be re-enabled at any time.

| Option | Type   | Required | Description                      |
| ------ | ------ | -------- | -------------------------------- |
| `name` | Choice | Yes      | `Moderation` or `Vanity Tracker` |

```
/module disable name:Moderation
/module disable name:Vanity Tracker
```

---

### `/module settings`

Display the current configuration for a module.

| Option | Type   | Required | Description                      |
| ------ | ------ | -------- | -------------------------------- |
| `name` | Choice | Yes      | `Moderation` or `Vanity Tracker` |

```
/module settings name:Moderation
/module settings name:Vanity Tracker
```

---

### `/module reset`

Factory reset a module. Clears all configuration, disables the module, and deletes any channels/roles that Caramel created during setup. Shows a confirmation prompt before executing.

| Option | Type   | Required | Description                      |
| ------ | ------ | -------- | -------------------------------- |
| `name` | Choice | Yes      | `Moderation` or `Vanity Tracker` |

```
/module reset name:Moderation
/module reset name:Vanity Tracker
```
