## Overview

The Caramel web dashboard lets you manage every aspect of your server's configuration from a browser — no Discord commands required. It connects directly to the same database as the bot, so every change takes effect instantly.

The dashboard is available at `/dashboard` after logging in with your Discord account.

## Logging In

1. Click **Login** in the navbar or visit `/login`
2. Click **Continue with Discord** — this opens Discord's OAuth2 authorization page
3. Authorize Caramel to access your account
4. You'll be redirected back to the dashboard

You need **Manage Server** permission on a guild for it to appear in your server picker.

## Server Picker

After logging in, you land on the server picker at `/dashboard`. This page lists all Discord servers where:

- You have **Manage Server** permission
- Caramel is currently a member

Each server card shows the server name, icon, member count, and number of active modules. Click a server to enter its dashboard.

If a server you manage doesn't appear, Caramel hasn't been added to it yet. Use the invite link to add it.

## Overview Page

The first page you see after selecting a server is the **Overview**. It gives you a snapshot of your server's current state:

- **Member count** — total members currently in the server
- **Active modules** — how many modules are currently enabled
- **7-day mod actions** — total moderation actions in the past 7 days
- **Active mutes** — members currently muted via the mute system

The overview also shows:

- A **7-day activity chart** — bar chart showing daily moderation action volume
- **Recent actions** — the 5 most recent moderation log entries (action type, user, moderator, timestamp)
- **Module status cards** — whether Moderation and Vanity Tracker are enabled or disabled

## Modules Page

Navigate to **Modules** in the sidebar to manage your server's modules.

Each module card shows:

- Module name and description
- Whether the module is currently enabled or disabled
- Quick stats (e.g. total warns logged, active mutes)

**To toggle a module:**

1. Click the toggle on a module card
2. The change saves immediately and syncs to the bot in real time

If a module's required configuration is missing (e.g. no log channel set), enabling it from the dashboard will prompt you to configure it first via Settings.

## Settings Page

Navigate to **Settings** in the sidebar to configure your modules.

### Moderation Settings

| Setting               | Description                                                                                                   |
| --------------------- | ------------------------------------------------------------------------------------------------------------- |
| **Log Channel**       | Text channel where moderation actions are posted. Select from a dropdown of all text channels in your server. |
| **Muted Role**        | Role assigned to muted members. Select from all server roles.                                                 |
| **Enable Thresholds** | Toggle whether warn counts trigger automatic escalation (mute/ban)                                            |
| **Mute Threshold**    | Number of warnings that triggers an automatic mute (default: 3)                                               |
| **Ban Threshold**     | Number of warnings that triggers an automatic ban (default: 5)                                                |

### Vanity Tracker Settings

| Setting            | Description                                                               |
| ------------------ | ------------------------------------------------------------------------- |
| **Status Keyword** | The keyword Caramel looks for in member custom statuses                   |
| **Vanity Role**    | Role assigned when the keyword is detected. Select from all server roles. |
| **Log Channel**    | Channel where vanity role assignment/removal events are logged            |

Click **Save** to apply all changes on the settings page. Changes take effect immediately and are synced to Redis so the bot reads the new configuration on its next operation.

## Logs Page

Navigate to **Logs** in the sidebar to view your server's full moderation history.

The logs table shows:

- **Action** — type of moderation action (WARN, MUTE, BAN, SOFTBAN, KICK, TIMEOUT, UNMUTE)
- **User** — Discord user ID of the moderated member
- **Moderator** — Discord user ID of the moderator who issued the action
- **Reason** — reason provided (if any)
- **Duration** — duration of the action (for mutes and timeouts)
- **Date** — when the action was taken

### Filtering and Search

- **Search** — filter by user ID, moderator ID, or reason text
- **Action type filter** — show only specific action types (e.g. only bans)
- **Pagination** — navigate through large log histories with server-side pagination

Logs are sorted newest-first by default.

## Sidebar Navigation

The dashboard sidebar includes:

- **Server switcher** — switch between servers without going back to the picker
- **Overview** — server stats and recent activity
- **Modules** — enable/disable modules
- **Logs** — moderation log history
- **Settings** — configuration for all modules
- **User avatar + logout** — your Discord avatar and a logout button at the bottom

## Dev Bypass Mode

If you're self-hosting and want to work on the dashboard UI without setting up OAuth or a real database, set `NEXT_PUBLIC_DEV_BYPASS=true` in `web/.env.local`. This skips authentication entirely and populates all dashboard pages with mock data. See the [Self-Hosting](/docs/installation) guide for details.
